CREATION D'UNE APPLICATION AVEC LARAVEL sous debian

INSTALLATION AVEC COMPOSER

  • Sur le terminal, à la racine du dossier où pointe le serveur, par exemple : /var/wwww/html/, on va créer un dossier qui contiendra notre application: mkdir NomDuDossier
  • on s'y rend en tapant: cd NomDuDossier
  • On crée notre application en tapant: composer create-project --prefer-dist laravel/laravel=5.3* NomDeApplication
  • s'y rendre avec cd NomDeApplication
  • Générer la clé de notre applicationen tapant sur le terminal: php artisan key:generate
  • On tape: composer update
  • Ensuite: php artisan serve (visualisation de la page d'acceuil de laravel= tout va bien avec localhost:8000 sur la barre de nav)

AJOUT DU COMPOSANT HTML

On se rend dans le fichier composer.json

  1. sous "laravel/framework": "5.3.*", ajouter après la virgule: "laravelcollective/html": "5.3.*"
  2. sur le terminal, taper: composer update

Dans le fichier app.php situé dans le dossier config:

  1. Ajouter dans la section des providers:

    2. 'providers'=>[.... collective\Html\HtmlServiceProvider:class, ...]

  2. Dans la section des alias: aliases: 'Form'=>Collective\Html\FormFacade::class,
  3. 'Html'=>Collective\Html\HtmlFacade::class,

AJOUT DU DEBUGBAR

Objectif: visualisation: d'une barre avec les erreurs eventuelles dans le code, du nombre de requêtes générées (les minimiser grâce à Eloquent)

  1. Sur le terminal taper: composer require barryvdh/laravel-debugbar
  2. Dans le fichier app.php du dossier config: dans la section des providers sous collective\Html\HtmlServiceProvider:class, Ajouter: Barryvdh\Debugbar\ServiceProvider::class,
  3. sur le terminal, taper: php artisan vendor:publish --provider="Barryvdh\debugbar\ServiceProvider"

AJOUT DE La langue de Molière, dossier fr (que vous trouverez dans ce repository sur github)

Dans le dossier ressource/lang, on rajoute le dossier fr

Fichier app.php du dossier config

'name'=>'NomDeApplication' 'local'=>'fr'

CONFIGURATION DU DRIVER POUR LES EMAILS

Objectif: créer un box mail virtuel: Pouvoir, en phase de développement, simuler l'envoi de emails.

Suivez les instructions sur ce lien

Dans le fichier .env situé à la racine de l'application: remplacer null par les codes obtenus après configuration de MailTrap

  1. MAIL-USERNAME=null
  2. MAIL-PASSWORD=null

CONFIGURATION DE LA BDD

Dans le fichier .env:

  • DB_DATABASE=NomDeBDD
  • DB_USERNAME=ex.root
  • DB_PASSWORD=mot de passe pour connection à PHPMysql
  • sur le terminal, taper: mysql -u root -p
  • Entrer son mot de passe
  • taper: create database NomDeBDD;
  • et ensuite show databases;

UNE COMMANDE INCONTOURNABLE:php artisan serve

Cette commande vous permettra de visualiser la page, il suffira de taper ensuite sur la barre de navigation: localhost:8000

Enfin se rendre à la page d'authentification du visiteur
